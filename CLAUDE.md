## 1. 프로젝트 개요

수시 원서 접수를 앞둔 수험생·학부모가 본인 성적(내신/수능)으로 지원 가능한 천안권 대학·학과를 조회하고, 판정(초상향~안정)을 확인하는 **무료·비회원·개인정보 무수집** 정적 웹서비스.

- **데이터 기준**: 2026학년도 입시결과 (대입정보포털 "어디가" 공개 자료)
- **1차 범위**: 천안 소재 4년제 8개교
- **핵심 UX 약속**: 성적 입력 → 결과 확인까지 **30초**, 회원가입 없음
- **톤**: "차분한 신뢰 + 또렷한 판정" (불안 자극 금지, 사실 서술)

## 4. 데이터 설계

> **⚠ 파이프라인 변경(확정)**: raw PDF가 전부 **이미지(텍스트 레이어 0)**로 판명 → 프로그래밍 파서 불가. **`scripts/render.mjs`(PDF→PNG) + `crop.mjs`(초밀도 표 좌우 크롭) + Claude 비전 전사**로 JSON 생성. 검증은 "사용자 원본 대조 + 자동 일관성검사(스키마·레코드수·판정분포)". DKU2 교과 판정분포 3/3/30/10 자동검증 통과로 방식 입증.
> **미제출 처리(확정)**: 컷/백분위 없는 '미제출' 모집단위는 records에서 **제외**하고 `meta.omittedNoData`에 건수만 기록.

### 4-1. JSON 스키마 (정답지 구조 준수)

**공통 meta**
```jsonc
{
  "meta": {
    "university": "단국대학교",
    "universityCode": "DKU2",
    "campus": "천안캠퍼스",        // 캠퍼스통합 대학은 명시
    "region": "천안",
    "year": 2026,
    "admissionRound": "수시",       // 수시 | 정시
    "screeningType": "교과",        // 교과 | 종합 | 정시
    "cutType": "70",                // 컷 기준(50/70)
    "referenceOnly": false,         // 종합전형 true → 판정 대신 "참고"
    "source": "대입정보포털 어디가",
    "note": ""
  },
  "records": [ ... ]
}
```

**records (수시)**
```jsonc
{
  "screeningName": "학생부교과우수자",  // 전형별 소제목 근거
  "dept": "국어국문학과",
  "field": "인문",                      // field-map.json 병합으로 채움
  "quotaFinal": 12,                     // 최종 모집인원
  "ratio": 8.5,                         // 경쟁률
  "waitlistFilled": 15,                 // 충원인원
  "score50": null, "score70": null,     // (있으면) 환산점수
  "cut50": 2.31, "cut70": 2.68,         // 등급 컷 (null 가능)
  "remark": null                        // "미제출 사유" 등
}
```

**records (정시) — 위 필드 + 추가**
```jsonc
{ "gun": "가군", "pct50": 78.5, "pct70": 74.2 }  // 군 + 백분위 컷
```

- **컷 null 규칙**: "미제출 사유" 행 → `cut* = null`, `remark`에 사유 텍스트
- **로딩 원칙**: 대학×구분별 JSON 분리 로딩 (전체 병합 파일 금지). 파일 추가만으로 대학 확장.

### 4-2. 계열 매핑 (`field-map.json`)
학과명 → 계열(`인문/자연/공학/보건의료/예체능/기타`) 매핑 테이블. 로더가 record의 `field`에 병합. 미매칭 학과는 `기타`로 두고 검수 시 보완.

### 4-3. 로딩 매니페스트 (`universities.json`)
대학 목록 + 각 대학의 구분별 JSON 파일 경로 + 캠퍼스 메타. 로더는 이 매니페스트만 읽고 필요한 JSON을 fetch.

## 5. 파서 설계 (`scripts\parse.js`)

### 파싱 유의사항 대응
| 문제 | 대응 |
|------|------|
| 표가 페이지 경계에서 잘림(학과명 분리) | 페이지 연속 병합 후 행 재조립 |
| 셀 줄바꿈으로 숫자 분리("92\n1.0\n4"→921.04) | 좌표 기반 셀 클러스터링으로 토큰 재결합 |
| "미제출 사유" 행 | 컷 null + remark 채움 |
| 전형별 소제목 | screeningName 근거로 캡처 |

### ★ 검증 게이트 (필수·차단 조건)
`scripts\validate.js`가 파서의 **DKU2 3개 산출물**을 `data\`의 **정답지 3개**와 대조:
- **레코드 수 완전 일치**
- **모든 컷 값 완전 일치**
- 불일치 시 → 파서 수정 후 재검증 (합격 전까지 7개교 변환 금지)

> **정답지 확보 방식(확정)**: 정답지 3개가 아직 없으므로 → **Claude Code가 DKU2 3종을 먼저 파싱해 `data\`에 JSON 초안 생성 → 사용자가 원본 PDF와 대조·수정하여 "정답지"로 확정** → 이후 파서를 이 정답지에 맞춰 검증. (사용자 승인 완료)

## 6. 판정 로직 (`js\verdict.js`)

컷70 기준. **초기값이며 검수 단계에서 조정 가능.**

### 수시 (내신 등급, 낮을수록 우수)
`diff = 내등급 − cut70`
| diff | 판정 | 색 |
|------|------|----|
| ≥ +1.0 | 초상향 | 빨강 #D64545 |
| +0.5 ~ +1.0 | 상향 | 주황 #E8883A |
| −0.5 ~ +0.5 | 적정 | 초록 #2E9E5B |
| < −0.5 | 안정 | 파랑 #3B82C4 |

### 정시 (평균백분위, 높을수록 우수)
`diff = cut70 − 내백분위`
| diff | 판정 |
|------|------|
| ≥ +7 | 초상향 |
| +3 ~ +7 | 상향 |
| −3 ~ +3 | 적정 |
| < −3 | 안정 |

- **종합전형**(`referenceOnly=true`): 판정 배지 대신 **"참고" 배지(보라 #8B7EC8)** + 정성평가 안내
- **컷 null 행**: "작년 데이터 없음(미제출)" 표시, 판정 제외
- **정시 등급 입력 시**: 구간 중간값으로 백분위 환산 + "백분위 입력이 더 정확" 안내

### 단위 검증 (개발 단계 필수 통과)
> 내신 3.00 → 단국대 교과 판정 재현: **초상향 3 / 상향 3 / 적정 30 / 안정 10** (마스터 기획서 명시값)

## 7. 화면 설계

### 7-1. 첫 화면 = 성적 입력(히어로)
- 서비스 소개 문단 없음. 상단부터 바로 입력.
- **수시/정시 탭** + 큰 성적 입력 필드(`inputmode=decimal`, 16px+) + 계열 선택(선택) + **[결과 보기]**
- 면책 고지: 입력 하단 한 줄 + "자세히 보기"

### 7-2. 결과 화면
1. **요약 스트립(상단)**: `안정 12 · 적정 30 · 상향 3 · 초상향 3` — 숫자 탭 시 해당 판정만 필터
2. **내 성적 sticky 칩**: `내신 3.00 기준` + [수정] (스크롤 중 항상 노출, 수정 시 입력 복귀)
3. **결과 리스트**: 기본 정렬 **적정→안정→상향→초상향**. 옵션: 컷순/대학명순/경쟁률순
4. **열람 모드**: 성적 미입력 시 빈 화면 대신 컷 데이터만 표시(입결 조회용)

### 7-3. 결과 카드(모바일 기준)
- **1줄**: 대학명 + 캠퍼스 뱃지 / 우측 판정 배지(색+텍스트)
- **2줄**: 학과명(굵게, 최대 2줄) + 전형명(보조, 정시는 군 표시)
- **3줄 — 시그니처 컷 스케일 바**: 컷50~컷70 구간 수평 바 + 내 성적 위치 점. 미입력 시 점 없이 구간만.
- **탭 → 아코디언**: 모집인원·경쟁률·충원인원·컷50/70 수치·미제출 사유
- **종합 카드**: "참고" 배지 + "서류·면접 중심 정성평가 전형으로 내신 컷만으로 판단할 수 없습니다"

### 7-4. 필터·검색
- 필터: 대학 / 전형구분 / 계열 / 판정등급 (모바일 = 하단 시트)
- 검색: 대학명·학과명
- **0건 화면 금지**: "조건에 맞는 학과가 없어요. 필터를 풀어보세요" + [필터 초기화]

## 9. 접근성·인터랙션 기준

- 판정 = **색 + 텍스트 병행**(색약 배려), 대비율 WCAG AA↑
- 터치 타겟 44px+, input 16px+, `inputmode=decimal`
- 모바일 필터 = 하단 시트, 애니메이션 150~200ms 1회. **장식성 모션 금지**, `prefers-reduced-motion` 존중
